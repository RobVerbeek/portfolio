Containerised Apache webserver

The goal of this project was to learn the power of Docker. To do so I had to research how Docker worked. I set up an Apache webserver that communicates with an SQL database, furthermore to prove my mastery of Docker I decided to add a Nginx loadbalancer to this project. The full documentation can be found at the bottom of this page.

Creating Images with a Dockerfile

The first step towards creating this project is the creation of the images that will be put inside docker containers. These images should be easily buildable by anyone. That's why a Dockerfile is used. These files also lets me customize what dependencies are installed on the file so that there are no errors if someone else wants to use it.

Containerizing the Application

To make the entire application modular and containerized, I made a docker-compose file. This file lets me use multiple images which have been built and add them together into a "stack". For this project my stack consisted out of an Apache, Nginx and SQL container.

I used nginx as a load-balancer for my application. The idea was to have multiple containers running the Apache service, which the load-balancer would then distribute the incoming traffic to. The goal of this is to ensure that the application stays available during high load times and there is less of a bottleneck. You can see the load-balancer in effect during the demo when the hostname changes once the page is refreshed.

Data Persistence with Volumes

Data persistence is important, and since containers will lose all data when they are rebooted or worst case scenario, they crash, measures must be taken in order to not lose any data.

To achieve this, I created volumes within Docker. These volumes will act as a storage space which won't be reset when a container goes offline. These volumes are created and linked to containers within the same docker-compose file that is used for the Docker-stack.
