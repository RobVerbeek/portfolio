CompeteMore

CompeteMore is a PHP based website which aims to give the students of Thomas More an easy-to-use platform for hosting and participating in competitions. To create this website I teamed up with four other students.

Designing Compete More

The project started off during project week, this week was dedicated entirely to working out a concept for the website, starting off from the documentation that was given to us. This contained use-case diagrams, use-case descriptions and low level prototypes. To make the best use of this information we first had a meeting with the client, where we asked questions to ensure we were on the same page.

Using the documentation and information from the meeting our team got to work at designing the website, reworking the use-case diagram, adding or removing use-cases as we deemed necessary and creating higher level prototypes. I was in charge of creating a new use-case-diagram. I kept the functionalities on which we agreed should be kept and added the new functionalities we thought should be added.

The Use-Case Diagram

The use case diagram displays all the functionalities, the people that will be able to access the application and the relationship between the users and which of the functionalities they have access to.

This is important to get right from the design stage so that you don't run into any issues while developing the application. Of course these ideas can still change during development but the diagram functions as a solid starting point.

During the concept discussions we came to the conclusion that the following roles will be involved in the application.

  • Guest(User): This is anyone who is looking at the website without being logged in, they should only be able to view the competitions and the results of finished competitions.
  • Student: Students are the core of this application, they will be the ones who create, join and vote for winners on the competitions.
  • Faculty Members: These are the teachers that work at Thomas More, they will also be able to vote on the winners.
  • Host: The person who created the competition, this can only be done by external members which are verified and students. The host is able to vote on the winner(s) and also flag invalid submissions from their competition.
  • Admin: They have a bunch of management functionalities and are the ones who approve a competition before it becomes public.

Developing Compete More

For the creation of this website we used Laravel, this includes the usage of Livewire and PHP.

Landing Page

I chose to create the landing page for our website, first creating a layout that everyone was happy with. To help with the creation of this, I decided to make use of Tailwind css and Flowbite. These speed up the process of making a layout considerably since I could start from premade components and alter them like I wanted.

On the technical side, I requested the important display information from the database such as name, description, thumbnail, status and various deadlines of the competition. All this information got neatly displayed on a compact card, then it was onto the functionality.

  • Different users (students, teachers, admins) have a different view of the page. For example only students are allowed to participate in competitions. So the registration functionality would only be visible to students.
  • Students or verified members can propose competitions which the admins will first have to accept before they are shown to the public.
  • During the registration phase students will be able to register.
  • While the competition is ongoing, participants will be able to submit a submission.
  • When the voting stage is active, all students on the platform can vote on a winner. But only if the competition has a public vote!
  • Pagination to avoid a massive scrollbar.

Competition Page

The natural follow-up to the landing page, this is where in-depth information of a competition gets displayed. The idea behind this page is almost identical to the landing page, but in more detail.

Some additional features include the following:

  • A timeline to show all of the deadlines
  • News & Announcements, Admins and the host of the competition will be able to create or delete announcements
  • Details such as the host, location and prize
  • Tips & Guidelines
  • A podium showcasing the winners once the competition has ended.

Submission Page

This is a simple page which will let participants upload a competition submission. They have to upload a thumbnail, and a file which corresponds to one of the allowed file types. If they want to, a comment section is available if they wish to share extra information about the submission.
